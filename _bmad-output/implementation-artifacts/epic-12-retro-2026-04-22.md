## What went well

**Research-first paid off twice.** Stories 12.3 (production create) and 12.6 (alerts manage) both had incorrect story-spec claims about IRIS API shape. The pre-dev Explore-agent research (reading `irislib/EnsPortal/Dialog/ProductionWizard.cls:181-189` for 12.3, `irislib/%SYSTEM/Monitor.cls` + `%Monitor/Manager.cls` for 12.6) revealed the real API and prevented the dev from burning time on non-existent methods. Epic 11 Rule #14 (prefer live probe over web research for IRIS-specific APIs) became operational practice, not just a doc.

**Epic 11 rules compounded.** Rule #3 (Config.* vs SYS.* vs Security.* class separation) directly guided BUG-4's fix shape — the dev applied the split without rediscovering the distinction. Rule #7 (REST handler I/O redirect + single-response dispatch) caught a potential issue during Story 12.6's compile flag choice (`k-d` to suppress display). First epic where `.claude/rules/project-rules.md` actively shaped implementation.

**Consolidated live verification in Story 12.4 worked.** One story exercised BUG-1/2/3/4/5 end-to-end AND re-ran the full Epic 11 regression matrix — zero regressions on the 16 prior bugs. Single-story gate model is cleaner than per-story live verify for epics with a shared bootstrap bump.

**Zero agent clarification requests across 14 dispatches.** Same pattern as Epic 11. The story specs (even when partially wrong) were rich enough that devs had everything they needed to act, and self-corrected when specs were empirically wrong.

**Tests grew meaningfully without bloat.** Net +~800 across packages (1117 → 1145 TS, +2 OS in Story 12.0). Each story's tests covered the actual bug surface plus forward cases. Story 12.6 over-delivered: +7 tests vs target +3.

**Rule feedback loop.** Every new Epic 12 bug produced either a rule candidate in deferred-work or a code comment pointing at an existing rule. Measurable evidence that Epic 11's meta-rule (#1) is working.

## What could've gone better

**Story specs contained false "X exists, don't touch it" claims twice.** Story 12.3 said `Ens.Config.Production.Delete()` existed (it doesn't). Story 12.6 said `acknowledge` would be a safe additive action (it has no native IRIS API). In both cases the dev discovered the error mid-implementation and widened scope unilaterally. Fix landed — but the pre-dev research that *did* happen on the *other* method (create in 12.3, the full action set in 12.6) missed these sibling concerns. **Rule candidate codified**: verify spec claims via live probe before trusting them (R3 below).

**BUG-6 unit tests couldn't catch the upstream property-extraction bug.** Story 12.4's `buildDocDbRestriction` translator is correct in isolation, and the unit test proved the restriction shape. But the tool is broken in practice because the JSON-to-typed-column extraction doesn't happen on insert — the lead caught this only by live-testing post-reload. **Lesson**: filter-translator unit tests need companion integration tests against the storage layer they target, or the bug class re-occurs in any similar wrapper.

**Server-reload dependency paused the pipeline twice.** Once for Story 12.1's first live test, once for Story 12.4's BUG-5/6 verification. Environmental, not a process flaw — but every TypeScript tool change is invisible to live testing until Josh reloads the MCP server. **Action**: add a CHANGELOG + contributor-docs note (A4 below).

**Two bootstrap bumps this epic — preventable.** Epic 11 had one consolidated bump; Epic 12 had two because Story 12.6's new handler method landed *after* Story 12.4's BOOTSTRAP commit. If 12.6's ObjectScript had been bundled into 12.4's commit, one bump would've covered the whole epic. Not a bug — the split arose naturally because 12.5 (TS-only, no bootstrap) sat between them — but worth a workflow note for future epics.

**CR for Story 12.6 edited an auto-generated file.** `packages/shared/src/bootstrap-classes.ts` is produced by `pnpm run gen:bootstrap`. The CR's ISO 8601 fix was correctly applied to `Monitor.cls` but *also* manually edited into the generated file. Lead had to regenerate the file to get the real hash (`974bbeab53a1`, not the CR's `f77518a0e09d`) and fix up references. **Rule candidate codified**: auto-generated files are output-only — never edit by hand, regenerate via the script (R5 below).

**Publishing backlog is aging.** Story 9.3 re-run and publishing-checklist items have been "pending" since Epic 8. Two consecutive bug-fix epics (11, 12) have produced a stable, well-tested codebase — the marginal value of one more self-testing pass is low compared to real-user signal from publish. **Team recommendation**: proceed to publish before Epic 13.

**Story file drift during active dev.** Linters / runtime reformatted story files between reads three separate times, producing tool-error retries. Friction, not blocker. Pre-existing; no action.

## Rules codified (Epic 12 → project-rules.md, now 18 total)

Four new rules added after Murat's risk-based triage — all pass the general-pattern-shape bar from Rule #1:

- **R1 (Rule #15)** — **Don't wrap method calls in `$Get()`** — `$Get(obj.%Get(…))` triggers multidim access on the receiver (crashes on `%DynamicObject`). Citation: BUG-3 at `Interop.cls:145,147`, Story 12.2.
- **R3 (Rule #16)** — **Verify story-spec "X exists" claims via live probe before trusting them** — story specs can be wrong about IRIS API shape; empirical verification precedes implementation. Citations: Stories 12.3 (`Ens.Config.Production.Delete` spec claim was wrong), 12.6 (`acknowledge` spec claim was wrong).
- **R4 (Rule #17)** — **`iris_doc_load` needs a glob-prefixed path** — bare file paths map class names wrong. Always use `src/**/File.cls` or `src/ExecuteMCPv2/**/File.cls`. Citation: recurring deploy bug caught three times across Epic 12 before being codified.
- **R5 (Rule #18)** — **Auto-generated files are output-only** — never hand-edit `bootstrap-classes.ts` (or any file produced by a script); regenerate via the script. Citation: CR for Story 12.6 edited the generated file directly, producing a hash that didn't survive the next regen.

Rules skipped (narrow or process-shape, not general pattern):
- **R2** — `ChangePassword` boolean vs `Password` setter — narrow one-property pair; Rule #2 ("read IRIS class source") already covers prevention.
- **R6** — One consolidated bootstrap bump per epic — process-shape, better as an epic-cycle workflow note than a rule; to be tracked in epic-cycle-log conventions.
