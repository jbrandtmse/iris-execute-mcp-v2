## What Went Well

1. **The dedicated research story (23.0) earned its keep — it caught a spec bug before any code depended on it.** The threshold-direction parenthetical wrongly implied journal/license/lockTable were "free% below" (descending) when they're ascending %-utilized; 23.0 caught it by checking the spec's own claims against live semantics, and folded `memory` out with evidence (`SYS.Stats.Dashboard` has no memory field). → **Rule #42.**
2. **QA caught a namespace-shape coverage hole that a green suite AND a green smoke both hid.** The `interop` gate-failure branch (`interopEnabled:false` when Ens classes are absent — the story's own §5-AC-4 highest-named risk) was untested because the dev's `%UnitTest` and live smoke both ran only on HSCUSTOM, which *has* `Ens.Director`. QA proved `%SYS` lacks it and pinned the branch with a real forced-failure test. → **Rule #41.**
3. **The dev satisfied Rule #34 for a `scope:"NONE"` tool with no namespace knob** by standing up a *disposable* SADEMO-bound webapp, proving `interop` is genuinely namespace-sensitive (0 queues/stopped vs 7 queues/running) while mirror/ECP are instance-wide, then deleting it — a genuine smoke instead of a residual-risk gap. → **Rule #40.**
4. **Clean carry-in discipline.** CR 22.1-3 folded into 23.1 (Monitor.cls already open + bootstrap re-rolling); CR 23.0-2/-5 implemented in 23.1; 7 prior CR items reconciled resolved across the epic.
5. **The pipeline model held.** Sonnet dev/QA + Opus review across all three stories; the Opus reviews adversarially re-verified load-bearing claims live (locks formula byte-matched `%Monitor.System.LockTable`; namespace correctness confirmed; verdict-engine directions confirmed) rather than reasoning about them.

## What Could've Gone Better (process, not people)

1. **The bootstrap *dual roster* surprised the dev mid-23.1.** Adding a new `.cls` required editing BOTH `gen-bootstrap.mjs`'s hand-maintained ordered `classes` array AND `bootstrap.test.ts`'s parallel roster — two lists, both "MUST stay in sync," not glob-discovered. Rules #18/#24 covered the regen *discipline* but not this two-places gotcha. → **Rule #39.**
2. **The fully-configured default namespace masks "not-configured" branches.** The same root that let the interop-gate branch ship untested (HSCUSTOM has Ens; mirror/ECP absent). Now guarded at the unit level by #41 and the smoke level by #40.

## Key Insights

1. **A spec's own claims deserve the same live verification as its API shapes.** The port author's mental model is exactly what's wrong when the spec is wrong — 23.0's job was to run the reference reality (live thresholds) and reconcile, not to trust the prose. (Rule #42, extending #16.)
2. **"Green suite + green smoke" is not "all branches covered" when a branch only fires in an un-configured namespace.** The default namespace's completeness is a blind spot at BOTH the test and smoke levels. (Rules #40 + #41, extending #34.)
3. **Generated ≠ single-source.** A generated artifact (`bootstrap-classes.ts`) can still have two hand-maintained inputs (the generator's ordered array + the drift test's roster). Regen updates content, not membership. (Rule #39, extending #18/#24.)
