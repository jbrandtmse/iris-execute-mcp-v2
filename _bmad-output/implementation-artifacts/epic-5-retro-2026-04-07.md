4. **Story 5.0 cleanup was the most thorough yet** — 14 action items resolved across TypeScript (logger, HTTP client, cursor, atelier, transport) and ObjectScript (PermissionCheck, tProps, password sanitization, property helpers). Codebase measurably cleaner.

## What Didn't Go Well / Challenges

1. **8 bugs found only through live/MCP testing** — Unit tests with mocked HTTP cannot catch IRIS API assumptions or cross-namespace issues. Step 2.5 (curl) caught 5 of these. The remaining 3 required actual MCP server connection testing which wasn't done until end-of-epic.

2. **IRIS API assumptions by dev agent** — 5 out of 8 stories assumed IRIS APIs existed that didn't (`GetAutoStart`, `Ens.Queue` SQL, `BusinessType` column, `AdapterClassName` column, `Config.MapPackages.Create("%ALL")`). The dev agent fabricates plausible-sounding API names.

3. **Pre-existing dual-JSON namespace bug (7 methods, 3 files)** — `Global.cls`, `Command.cls`, `UnitTest.cls` all called `RestoreNamespace()` outside try/catch after response rendering. When operating in a switched namespace, `ExecuteMCPv2.Utils` was invisible, causing a second error JSON appended to the response. This existed since Epic 3 but was never caught because cross-namespace testing wasn't done.

4. **ValidateGlobalName too strict** — Rejected dotted global names like `Ens.AutoStart`. The regex `.AN` only allows alphanumeric, but IRIS globals commonly use dots. Pre-existing since Epic 3.

5. **IPM module.xml not kept current** — Missing Config.cls, Security.cls, and Interop.cls. Three epics of handler classes not registered in IPM. Caught during post-epic review.

## Lessons Learned

1. **Curl testing is necessary but not sufficient** — Curl tests the IRIS REST layer (wrong APIs, missing columns, namespace switching). MCP tool testing through the TypeScript HTTP client catches response parsing, dual-JSON, content-type, and cross-namespace I/O redirect issues. Both layers required.

2. **Cross-namespace testing must be explicit** — The dual-JSON bug and I/O redirect `<NOROUTINE>` existed for 3 epics because nobody tested tools with `namespace=OPTIRAG`. One cross-namespace call per tool would have caught both.

3. **%ALL namespace must be created before mappings** — `Config.MapPackages.Create()` validates the namespace exists. `%ALL` is a virtual namespace that must be explicitly created first with `Config.Namespaces.Create("%ALL", ...)` using `%DEFAULTDB` for globals/routines.

4. **Dev agent fabricates IRIS APIs** — Research-first (Perplexity, class dictionary SQL queries) is essential for Ens.* and %SYS.* classes. Story context files should include verified API references, not assumptions.

5. **IPM module.xml must be updated in the same story that adds handler classes** — Not as a post-epic catch-up.
