## What Went Well

1. **The Opus review layer caught the highest-stakes defects — on a pure data file.** Story 24.0 is just a `Record<string,"read"|"write">`, yet the review found TWO HIGH safety misclassifications (`iris_oauth_manage:discover`, `iris_transform_test` marked `read` when they persist/execute), each *verified against live IRIS source* (`%SYS.OAuth2.Registration.Discover` commits a txn; DTL executes arbitrary code). A false `read` there punches straight through read-only mode. → **Rule #44.**
2. **Mechanical back-compat discipline held.** 24.1 threaded the preset as optional default-`undefined` params (Epic 20 F2 pattern); the unset-preset capstone deep-equals the pre-change policy over the full 141-key universe; unset SQL caps produce no `rowsCapped`/no timeout arg (conditional-spread absent-key shape). Rule #19 proven, not asserted.
3. **QA added orthogonal value every story and made a latent gap visible.** 24.1: cross-surface agreement tests (reported policy == enforced decision across resource/gate/discovery). 24.2: a test *demonstrating* the `mergeProfile` caps-propagation gap so the review couldn't miss it.
4. **The live smoke proved the headline for real, not by proxy.** Built dist over MCP stdio; the server logs themselves proved "changes nothing" (a refused write emitted only a GET, no PUT).

## What Could've Gone Better (process, not people)

1. **Story 24.1 shipped the preset engine with zero docs.** The docs obligation lived only in 24.2's AC, so a 24.1-only build had a live, security-relevant `IRIS_GOVERNANCE_PRESET` documented nowhere. The 24.2 dev caught it and folded the rollup in — but that was luck-of-the-pipeline, not a gate. → **Rule #43.**
2. **Two real gaps were caught by review/smoke, not dev tests:** the `mergeProfile` caps-propagation gap (caps silently ignored on named profiles — the exact "point it at *production*" scenario) and `IRIS_SQL_TIMEOUT="Infinity"` → `setTimeout` clamps to ~1ms → silently disables SQL. Both fixed before ship by the adversarial layers.
3. **The Story-24.0 AC over-broadened the read-verb allowlist** (added `test`/`find`/`search`/etc. beyond the binding spec's `list/get/view/status/check/history/explain/stats`), creating the `iris_transform_test` spec-divergence the review flagged and the lead ratified.

## Key Insights

1. **A hand-curated safety map needs a mechanical cross-check against an independent declared signal — completeness + human review + value-pins are blind to which SIDE each key lands on.** `readOnlyHint` is that signal for read/write. (Rule #44, extending #20.)
2. **A feature story should be independently doc-complete for what it ships; the closing rollup enriches, it doesn't first-document.** Deferring 100% of docs leaves an interim build with an undocumented safety feature. (Rule #43, pairing with #30.)
3. **The adversarial review + live smoke are where governance-safety correctness is actually established** — three of the epic's most important fixes (2 misclassifications, the caps-propagation gap, the Infinity timeout) came from those layers, not from green dev-stage tests.
