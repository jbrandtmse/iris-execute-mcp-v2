# Story 27.1: Remaining Diff Domains + Redaction

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As an operator comparing two IRIS profiles with `iris_env_diff`,
I want the diff to also cover **mappings, System Default Settings, web applications, and system config** (not just documents), with credential-ish setting values redacted,
so that I can detect the configuration drift that lives OUTSIDE git — the surfaces real IRIS shops actually diverge on — without leaking secrets.

This is Story 2 of Epic 27. It extends the existing `iris_env_diff` tool (Story 27.0 shipped its `documents` domain) with the four remaining domains, reusing the EXISTING custom-REST endpoints each domain's tool already calls. **TS-only — no ObjectScript / bootstrap change** (all four endpoints already exist and are deployed).

## Acceptance Criteria

1. **AC 27.1.1** — `mappings` (full tuples incl. subscript-level), `defaultSettings`, `webapps` (curated property subset, instance-specific paths excluded by default), `config` domains implemented against the existing endpoints per profile; unit tests with fixture payloads for every domain's three buckets.
2. **AC 27.1.2** — SDS values whose setting name matches `password`/`secret`/`key` (case-insensitive) reported as `[REDACTED:differs]`/`[REDACTED:identical]` — values never leave the server in diff output (spec §6 AC 6).
3. **AC 27.1.3** — `summary.driftCount`/`identicalCount` roll up across domains; `domains` param filters correctly.

### Integration note (lead-side gate)

This story EXTENDS the existing `iris_env_diff` tool (no NEW service/component introduced) — its ACs are self-verifying (fixture-tested three buckets per domain produce observable diff output). The diff output's first consumer is Story 27.2's `promote:plan` generator; 27.1's fixture tests + the roll-up/filter ACs are the integration proof for this story.

### Redaction (AC 27.1.2) — output-only, not wire

"Values never leave the server **in diff output**" = the returned `structuredContent`/`content` must not contain a credential plaintext. The tool DOES receive the value from IRIS (same MCP-server trust boundary, like every other field), compares source vs target internally, then emits ONLY `[REDACTED:differs]`/`[REDACTED:identical]`/`[REDACTED]`. Server-side redaction (in `Interop.cls`) is NOT required and would be an ObjectScript/bootstrap change (out of this TS-only story's scope). Match on the `setting` NAME (case-insensitive substring of `password`/`secret`/`key`) — NOT the value.

### One-instance test shape + Story 27.4 capstone (IMPORTANT — resolves the 27.0 finding)

Story 27.0's re-smoke found HSCUSTOM and SADEMO SHARE code databases → `documents`/`webapps`/`config` are IDENTICAL between them on one instance (residual risk). BUT **`mappings` and `defaultSettings` are genuinely per-namespace** — HSCUSTOM and SADEMO have different mapping tables + SDS tables. So **real cross-namespace drift IS seedable via the `mappings` or `defaultSettings` domain**, which reconciles the Project Lead's "drift is seedable" answer (it holds for these two domains, just not `documents`). **Story 27.4's capstone (AC 27.4.2) should seed its drift via a `defaultSettings` (SDS) or `mappings` change** (namespace-scoped), NOT a class edit (shared code DB). Flag this in the 27.1 dev notes for the 27.4 story.
