## What Went Well

1. **Clean dependency-ordered build, zero rework.** 14.1 profiles → 14.2 `server` param → 14.3 governance engine → 14.4 enforcement → 14.5 resource + capstone → 14.6 docs. Each story consumed the prior's seam exactly as specified; the de-risking-priority call (per-profile session isolation, landed first in 14.1 AC 14.1.6) held all the way to the 14.5 cross-server capstone.
2. **Back-compat discipline as the spine.** Every story carried a release-critical "off = today's behavior" gate with a *mechanical* proof, not a prose claim (`loadConfig` byte-for-byte unchanged; 141-key baseline all-enabled under empty config; gate pure pass-through; additive schema/capability). → codified as **Rule #19**.
3. **Provable back-compat via the generated governance baseline (D3).** `gen-governance-baseline.mjs` made "no existing capability disabled by default" mechanically verifiable rather than a hand-maintained flag — the standout design idea of the epic. → codified as **Rule #20**.
4. **Adversarial code review earned its keep — caught 3 real bugs the (passing) tests missed:** 14.1 default-profile `bootstrapAttempted` flag (HIGH; would have broken 14.2's seam), 14.3 prototype-member governance-key leak (MED; broke the `getEffectivePolicy` length invariant the gate relies on), 14.5 capstone-mock `getSetCookie` (MED; would have flaked the D1 isolation capstone on the Node-18 floor). The 14.4 enforcement chokepoint passed all three adversarial layers clean.
5. **The capstone was treated as the epic-done gate and verified genuine.** AC 14.5.6 was named as the gate, kept in the default suite (not `.integration.test.ts`), and review confirmed it actually exercises real per-instance cookie jars + the shared gate. → codified as **Rule #21**.
6. **Lead smokes against the built artifact + live IRIS.** Each story's smoke ran the built `@iris-mcp/shared` in a real process — confirming the wired-up system end-to-end where the (correct) unit tests used mocked fetch. → codified as **Rule #22**.
7. **Empirical SDK/IRIS verification (Rules #14/#16) throughout.** Devs probed Zod `.options` introspection, the MCP SDK 1.29 resource API, and the SDK's `isError`-skips-output-validation behavior before coding — no "the API probably looks like…" guesses.

## What Could Be Better (watch-items)

1. **The `mutates`/new-action seed machinery is built but unexercised by real tools.** Epic 14 added no new IRIS tools, so the seed/classification path is proven only by synthetic fixtures. Its first REAL exercise is Epic 15's first governed write tool — latent risk that a real-tool edge wasn't caught.
2. **A deferred item was deferred again toward Epic 15** (14.4 CR: `computeGovernanceKey` + the baseline generator would mis-handle a future `action` enum wrapped in `.optional()`/`.default()`/`.nullable()`). It is in `deferred-work.md` — the Epic 15 Story 15.0 triage MUST pick it up before/with the first governed write tool.
3. **Pre-existing doc drift surfaced** (migration-v1-v2.md dotted v2 tool names from the Epic-9 flattening; architecture.md stale `(16 tools)`/`86 tools`). Correctly left out of Story 14.6 scope; logged in `deferred-work.md`.
4. **Windows/libuv teardown noise** in lead smokes (`process.exit` with open keepalive sockets → benign `UV_HANDLE_CLOSING` assertion after the pass line). Cosmetic; noted in Rule #22 so it isn't mistaken for a defect next time.
