## What Went Well

- **F2 shipped a genuinely-hard requirement honestly.** `clean` is a destructive write that must be callable out of the box. The Project Lead rejected the misclassify-as-read shortcut; the dev built an orthogonal `defaultEnabled` marker threaded as a **default-empty** param through `defaultSeed`/`effective`/`getEffectivePolicy`, so `clean` ships enabled while `mutates:"write"` stays truthful and the frozen baseline stays untouched. The empty set is byte-for-byte the prior seed for the other 4 servers. Codified as **Rule #32**.
- **Adversarial code review caught a silent-misconfiguration hole.** Both the Blind and Edge hunters independently flagged that a `defaultEnabled` action not declared as a per-action `write` in `mutates` would silently ship DISABLED. Auto-fixed with a registration-time fail-fast cross-validation (+4 tests) — a class of bug that can't exist now.
- **QA's mutation-tested capstone** proved the all-other-writes-still-disabled invariant is real: inject `defaultEnabled:["add"]` on a sibling → red → revert → green + git-clean. Genuine test, not a tautology (Rule #21 shape).
- **Source-grounded story (Rule #16) surfaced a latent bug.** Writing the spec against the actual `irislib/Ens/Director.cls` exposed that `RecoverProduction()` takes NO args, so the existing `recover` action's `RecoverProduction(tForce)` call was a latent `<PARAMETER>` defect — fixed in-scope and live-verified.
- **`^Ens.AppData` research made the guard meaningful.** The double-gate refusal names the actual consequences (HL7 sequence numbers, file/FTP done-file re-ingestion, batch/control state) — verified live over HTTP that the message renders.

## What Was Challenging

- **`SanitizeError` silently strips caret-globals.** The dev's first double-gate message wrote `^Ens.AppData` and it came out blanked after sanitization, gutting the consequence. Reworded to `Ens.AppData` (no caret); the TS description retains the caret (not sanitized there). Recurring-shape — any handler error message naming a global hits this. Codified as **Rule #33**.
- **Framework-foundation blast radius.** F2 changed 3 shared signatures used by all 5 servers. Mitigated by the default-empty param + the capstone test, but it is exactly the kind of change that needs a genuine cross-server invariant test (which it got).
