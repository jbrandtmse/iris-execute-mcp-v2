## What Went Well

1. **Back-compat proven mechanically (Rule #19).** The no-prompts capability snapshot deep-equals today's `{tools, resources}` — the SDK advertises `prompts` only when a prompt is registered, so an empty pack is byte-for-byte unchanged. `toEqual`, not prose.
2. **The Epic-24 preventive net (Rule #44) landed and held genuine.** CR 24.0-1's `readOnlyHint` cross-check shipped, was mutation-tested (removing a justification → red), and surfaced 15 divergences — all verified per-key as legitimate multi-action-tool cases (7 route through one `ctx.http.post` distinguished only by an `action` body field, needing an OS-side cross-check). **Zero reclassifications** — no new latent misclassifications existed.
3. **The lead smokes caught real SDK-contract defects the unit tests could not.** CR 25.0-1 (zero-arg prompt rejected an omitted `arguments`) was FIXED; CR 25.1-6 (the same for all-optional-arg prompts) was diagnosed to a hard SDK limitation and DOCUMENTED. The per-server unit tests used `arguments: {}` and were blind to the omission edge. → **Rule #46.**
4. **Rot prevention is mechanical.** `validate-prompts` (102-name surface) + `gen:skills --check` + the doc-rot sync guard all run in the default suite — a tool rename or doc drift breaks CI, not users.

## What Could've Gone Better (process, not people)

1. **CR 24.0-1 was routed to the wrong story by the prior retro.** The Epic 24 retro sent the `readOnlyHint` cross-check to "Story 25.0", but it needs leaf-package annotations that `@iris-mcp/shared` cannot import (circular). Caught at the retro-review gate and moved to 25.1 (Rule #16 shape). The recurring structural lesson became **Rule #45**.
2. **The all-optional-arg omission (CR 25.1-6) only surfaced in the lead smoke.** A stricter unit test (omit `arguments`, not `{}`) would have caught it earlier — now encoded as the Rule #46 discipline.
3. **The Story-25.1 code-review stage backgrounded its review layers and returned early.** A harness/process hiccup; recovered by resuming the agent synchronously and collecting the layers' findings. Not a code issue.

## Key Insights

1. **Dependency direction dictates test placement.** A check that needs every package's metadata belongs in the all-packages aggregator, never in the shared foundation it all depends on. (Rule #45.)
2. **A new protocol capability must be smoked through a real MCP client, and the omitted-optional-field edge is where SDK arg-parsing contracts break.** Mocked/`{}`-arg tests structurally miss it. (Rule #46.)
3. **The adversarial review + lead smoke remain where correctness is actually established** — every one of this epic's material fixes came from those layers, not from green dev-stage tests.
