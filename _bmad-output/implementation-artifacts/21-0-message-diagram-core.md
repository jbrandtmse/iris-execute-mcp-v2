## Story

As an **integration engineer**,
I want **a tool that returns a Mermaid sequence diagram for a message-trace session**,
so that **I can see how messages flowed through the production — who called whom, sync vs async, request vs response, and where it errored — without opening Visual Trace**.

### Amendment A1 — dynamic SQL, not embedded SQL

`gen-bootstrap`'s class set deploys through `Setup.cls` into whatever namespace a server bootstraps — including **non-interop namespaces** where `Ens.MessageHeader` does not exist. Embedded SQL would add a compile/first-run schema dependency the bootstrap must not carry; every existing handler (see `MessageTrace`) uses `%SQL.Statement` + `?` params. Use `%ExecDirect` or prepared `%SQL.Statement`. The ORDER BY fallback: attempt the `TimeCreated, ID` statement; if prepare/execute reports SQLCODE < 0, retry `ORDER BY ID` (encode as two statement strings — cheap and explicit).

### Amendment A2 — gen-bootstrap.mjs is an ordered array, NOT a glob

[`scripts/gen-bootstrap.mjs:11-25`](../../scripts/gen-bootstrap.mjs) holds an explicit dependency-ordered `classes` list (Utils → Setup → handlers → Dispatch last). The proposal's "picked up by the glob" claim is wrong (Rule #16 discrepancy recorded). Insert the five Diagram entries between `Setup.cls` and the handler block so they compile before `REST/Interop.cls` (whose new handler references the facade). Editing the GENERATOR is correct; editing `bootstrap-classes.ts` is forbidden (Rule #18). Note: `src/ExecuteMCPv2/Tests/*.cls` are NOT in the array — keep it that way.

### Handler pattern (mirror, don't reinvent)

[`Interop.cls` `MessageTrace()`, lines 985-1066](../../src/ExecuteMCPv2/REST/Interop.cls#L985) is the exact template: `Set tSC = $$$OK` / `Set tOrigNS = $NAMESPACE` / Try; `$Get(%request.Data("param",1))` reads; count clamp; optional `SwitchNamespace(tNamespace, .tOrigNS)` with error render + `Set tSC=$$$OK Quit`; `Set $NAMESPACE = tOrigNS` immediately before EVERY `RenderResponseBody`; Catch restores namespace first line then renders `SanitizeError(ex.AsStatus())`; `Quit $$$OK`. NO `New $NAMESPACE` (breaks class resolution in catch blocks). Exactly one `RenderResponseBody` per control path (Rule #7). sessionIds CSV parse: `$Length(tCsv,",")` loop, each piece must be a positive canonical integer (`tPiece = +tPiece` && `+tPiece > 0`), count 1–20 else reject with a clear message.

### I1 sanitization recipe (Rule #33)

`ErrorStatus` is a `%Status`. One-liner: `Set tErrText = $System.Status.GetErrorText(##class(ExecuteMCPv2.Utils).SanitizeError(tErrorStatus))` — `SanitizeError` strips caret-global refs and internal details, and (Rule #8) the leading `ERROR #N:` prefix is handled; then **collapse CR/LF/TAB to single spaces** and truncate to a sane length (~200 chars) before it enters a `%%` comment or `warnings[]`. Never author error text that names a global with a caret. If `IsError=1` but `ErrorStatus` is `$$$OK`/empty, emit the `[ERROR]` suffix with a generic `message reported error` summary.
