## Coverage gaps (deliberate, per strictly-additive constraint)

1. **Vanished-foundation-key failure branch (`--check` → exit 1).** The inverse of
   test (a). Automating it would require mutating the real built server dists (to
   drop a frozen key) or stubbing the committed frozen baseline — both forbidden by
   the strictly-additive constraint (cannot touch `governance-baseline.ts`; cannot
   perturb dists/source). The branch IS exercised by the authoritative one-directional
   drift logic mirrored in `governance.test.ts` ("every FROZEN foundation key still
   exists … one-directional"), and the `--check` mode is a CLI mirror of that same
   logic. Documented as a gap rather than faked via dist mutation.
2. **`--force` write path.** AC 16.0.3 restores today's write behavior under `--force`.
   Not automated because any test driving the real write path would overwrite the
   FROZEN `governance-baseline.ts` (regrowing it to the live surface); even with
   before/after restore, a mid-test crash could leave drift in the frozen file —
   an unacceptable risk against the additive/frozen constraint. The refusal-without-`--force`
   path (the security-relevant footgun guard) IS covered (test 3).
