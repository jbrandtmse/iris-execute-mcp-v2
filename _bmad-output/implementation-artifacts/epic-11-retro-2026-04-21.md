## What went well

1. **Epic 11 ran end-to-end with effectively zero human interventions.** Nine agent dispatches (sprint-planning + 4 dev + 4 CR) produced 9 commits and resolved 16 bugs. Exactly one user interaction (a checkpoint the lead offered deliberately after Story 11.1, answered `continue`). Zero clarification requests from agents back to the lead. Attributable to:
   - Rich story specs with root-cause hypotheses, exact file:line references, and fallback paths documented up-front.
   - Embedded live-verification ACs — each story had a concrete reproduction checklist the dev and CR could execute without lead guidance.
   - The `irislib/` / `irissys/` local exports: dev agents cross-checked IRIS APIs directly instead of guessing.

2. **Single-BOOTSTRAP_VERSION-bump strategy delivered cleanly.** Stories 11.1, 11.2, 11.3 all touched ObjectScript; only Story 11.3 ran `npm run gen:bootstrap`. One hash change, one auto-upgrade event per install. Planning decision in the Sprint Change Proposal paid off.

3. **Live verification at THREE layers caught everything before merge.** Dev verification on HSCUSTOM, lead verification on USER (cross-namespace), CR cross-checked with `git diff` on `bootstrap-classes.ts`. All 12 ObjectScript bugs resolved cleanly at all three layers. TS-only Story 11.4 correctly fell back to unit-tests + baseline-reproduction pattern (same as Story 10.6).

4. **Code review caught real things without blocking.** Zero HIGH, zero MEDIUM, one LOW auto-resolved (ssl.test.ts fixture drift in 11.4 — stale `protocols: 24` from pre-Story-11.2 state). Reviewers validated specific risks flagged in story files:
   - CR 11.2 confirmed `$Piece(tUserRoles, ",", tI) = "%All"` exact-equality loop (not substring match) — no false-positive on names like `%AllCustom`.
   - CR 11.3 verified `BOOTSTRAP_VERSION` hash via `git diff packages/shared/src/bootstrap-classes.ts` and confirmed deterministic regeneration.
   - CR 11.4 validated `horologToIso` edge cases (null, undefined, empty, malformed) all return `""` without throwing.

5. **Perplexity → live-probe pivot was decisive.** Research returned irrelevant results twice (Bug #9 system-wide counter API, Bug #10 `%All` permission semantics). Dev pivoted to `iris_doc_list SYS.Stats%` + `iris_doc_get SYS.Stats.Global.cls` both times and found the authoritative API in under five tool calls. Now codified as Rule #15.

6. **Pre-release SSL break landed cleanly.** `protocols` (deprecated IRIS property, never wired through) → `tlsMinVersion` + `tlsMaxVersion` in one server-side-then-client commit pair (11.2 then 11.4). No compatibility shim — correct call because `protocols` never worked. Clients that had written `protocols: 24` had been getting silent drops anyway.

7. **Inline CHANGELOG + per-story README updates meant no docs rollup story.** Epic 10 needed Story 10.3 because two new tools landed (tool count 21 → 23). Epic 11 added zero tools → no bulk doc edit needed → zero stories of pure doc work. The whole documentation delta lives in git history commit-by-commit, which is easier to review than a monolithic rollup commit.

8. **Story 11.3 live-verification gate protected Stories 11.1 and 11.2 too.** Because the bootstrap bump was deferred to 11.3, its AC 11.3.5 required re-running all 12 Epic 11 ObjectScript bug reproductions post-bump. If any story's fix had regressed when combined with the others in the single bump, it would have surfaced there. None did.

9. **Bug #15 surprise caught a misconception.** Story 11.3 predicted instance locale was `araw` (Arabic-Windows) based on the `خطأ` error prefix. Actual locale is `enuw` (English-Windows); the Arabic comes from NLS message tables, not locale. Dev correctly separated the two concerns when implementing the fix. Now codified as Rule #13.

## What could have gone better

1. **Retrospective lessons weren't being codified into durable rules — the biggest finding of this retro.** Epic 10 retro noted "Atelier asymmetries are first-class concerns" and "read class source when wrapping". Those lessons never made it into a `.claude/rules/` file. Epic 11 then re-encountered the exact same pattern in Bugs #3, #4, #5, #6, #9 — six bugs from root causes Epic 10 already had seen. Fix: **new [`.claude/rules/project-rules.md`](../../.claude/rules/project-rules.md) file landed as part of this retro, with Rule #1 being the meta-rule that enforces codification at every retrospective**. Eight of Epic 11's 16 bugs would have been prevented by the rules now in place.

2. **Perplexity MCP returned irrelevant results twice.** Bug #9 counter API: asked for `%Monitor.System.Globals` vs `SYS.Stats.Globals` accessing instance-wide counters → got results about the English word "inter". Bug #10 `%All` semantics → similar junk. Not blocking (pivot to live probe worked), but worth flagging that Perplexity is unreliable for IRIS-specific API questions. Now codified as Rule #15 — prefer live IRIS probe for IRIS-specific APIs.

3. **Story 11.4 TS-only live-verification gap is a known limitation.** Running MCP server processes use the pre-11.4 compiled code until restart. Dev correctly verified via unit tests + baseline-bug-reproduction (bug still reproduces in running server, proving baseline; unit tests prove fix in new code). This is the standard TS-only pattern (matches Story 10.6). Next client reconnect activates the new code. Not a regression — just an observable gap.

4. **One small fixture drift pre-existed from Story 11.2.** `ssl.test.ts` had two mock rows still using `protocols: 24` after Story 11.2's server-side break. CR 11.2 didn't catch it because the test assertion didn't touch `protocols` — the drift was in the mocked response payload, not in the assertion. CR 11.4 caught it and auto-fixed. Lesson for future CR: when a schema changes, grep for ALL references to the old field name in tests + mocks, not just assertions.

5. **Lead Step 2.5 verification for Story 11.4 was weak.** The running MCP server still had Story 11.3's binary, so my direct tool calls (`iris_doc_search`, `iris_rest_manage`, `iris_analytics_cubes`) confirmed the old behavior — which is expected for TS-only changes but made lead verification less rigorous than Stories 11.1–11.3. I accepted the dev's baseline+unit-test verification instead. Future TS-only stories should build the verification limitation into the plan from the start rather than discovering it at Step 2.5.

## Lessons learned (carry to whichever epic comes next)

1. **Retrospectives produce rules, not just prose.** Every retro ends with a "Rules to codify" step that appends to [`.claude/rules/project-rules.md`](../../.claude/rules/project-rules.md). Narrow one-off fixes stay in the retro document; class-of-bugs lessons become rules. Already codified as Rule #1 (the meta-rule).

2. **When wrapping an IRIS system class, read the source first.** `List` ROWSPEC ≠ full property list. `Config.*` carries configuration but not runtime state. `[Deprecated]` properties may be SELECT-able but disconnected from runtime. Cheap to check, expensive to miss. Now codified as Rules #2, #3, #4.

3. **`$ZU(...)` built-ins are usually per-process.** For instance-wide metrics, use `SYS.Stats.*` / `$SYSTEM.Monitor`. Cross-check against the IRIS SMP Dashboard. Now codified as Rule #5.

4. **`%All` is a super-role — short-circuit in permission walks.** Any handler walking `Resources` strings must special-case `%All` holders. Use exact `$Piece` equality, not substring match. Now codified as Rule #6.

5. **REST handler error envelopes need I/O restore + single-response-dispatch.** `ReDirectIO(0)` + `Use tInitIO` (no mnemonic arg) before `RenderResponseBody`. Flag-plus-post-catch pattern guarantees exactly one response per request. Now codified as Rule #7.

6. **`SanitizeError` must strip existing `ERROR #N:` / `خطأ #N:` prefixes** before re-wrapping. Otherwise every error is doubled. Now codified as Rule #8.

7. **Don't swallow `%Status` on error paths.** Propagate via `SanitizeError(tSC)`; don't wrap in generic `"Failed to X for Y"` strings that drop diagnostic signal. Now codified as Rule #9.

8. **MCP tool defaults must be wire-explicit.** If the Zod schema advertises a default, send it every time — don't rely on "caller omits → server defaults to same value." Now codified as Rule #10.

9. **`$HOROLOG` values must be converted to ISO 8601 in the TypeScript layer.** Preserve raw in `<field>Raw` for debugging. Round-trip verify one value against IRIS `$ZDATETIME`. Now codified as Rule #11.

10. **`%REST.API.GetAllRESTApps` is spec-first by design.** Any "list REST applications" tool must offer both `spec-first` (default) and `all` modes. Now codified as Rule #12.

11. **IRIS instance locale is NOT the same as NLS message translation tables.** Don't infer one from the other. The `خطأ` error prefix comes from loaded message tables, not the `%SYS.NLS.Locale` setting. Now codified as Rule #13.

12. **Password redaction in error messages must gate on `$Length(password) >= 8`.** Short candidates corrupt unrelated English words. Now codified as Rule #14.

13. **For IRIS-idiosyncratic APIs, prefer live probe over web research.** Perplexity struggles with deep IRIS internals; `iris_doc_list` + `iris_doc_get` against the running instance is faster and authoritative. Now codified as Rule #15.

14. **Single-BOOTSTRAP_VERSION-bump per epic (with bump gated at the last OS-touching story) is the correct pattern** when multiple stories change ObjectScript. One auto-upgrade event per install, cleaner commits, easier rollback. Keep using it.

15. **Inline CHANGELOG + per-story README updates scale better than end-of-epic docs rollups** when net new tools = 0. Skip the docs story unless the tool count changes.

## Rules codified

See [`.claude/rules/project-rules.md`](../../.claude/rules/project-rules.md) for the full rule set. This retro added **14 rules** (15 initially drafted, 1 removed during self-audit — see note at end):

1. **Meta** — codify retrospective lessons as rules
2. **Read IRIS class source before wrapping**
3. **Config.* vs SYS.* vs Security.* class separation**
4. **`[Deprecated]` property caution**
5. **`$ZU(...)` per-process vs instance-wide**
6. **`%All` super-role short-circuit**
7. **REST handler I/O redirect + single-response dispatch**
8. **`SanitizeError` prefix hygiene**
9. **Error propagation — don't swallow `%Status`**
10. **Wire-explicit defaults in MCP tools**
11. **`$HOROLOG` → ISO 8601 in TypeScript layer**
12. **Spec-first vs all REST application listing**
13. **IRIS locale vs NLS message tables**
14. **Prefer live IRIS probe over web research for IRIS-specific APIs**

**Self-audit (removed during review against intent criteria):** an initial Rule #14 *"Password redaction — gate on length"* was drafted to cover Bug #8. During a re-read of the rule set against the frontmatter + Rule #1 intent criteria, it was removed because Murat's risk-based triage in this retro's own "Rules to codify" table had already classified Bug #8 as **narrow** (*"not a general pattern — fix is in code, no rule needed"*). Codifying it anyway would have violated Rule #1's *"narrow one-off fixes do NOT become rules"* principle. The fix remains in the code (`src/ExecuteMCPv2/REST/Security.cls` validate branch, commit `b3be8a4`) and in this retro's bug log. If a future epic adds a second handler that does substring redaction of short user-supplied values and hits the same class of bug, that would be the moment to codify it as a rule.
