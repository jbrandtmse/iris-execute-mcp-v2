# Story 27.0: Profile Client + Document Hashes + Documents Diff

Status: done

## Story

As a developer/operator running the IRIS MCP suite against two configured profiles (e.g. `stage` and `prod`),
I want to compare the **ObjectScript source** of both profiles by content hash in a single call,
so that I can detect code drift across environments without the manual export/diff dance IRIS shops do today.

This is Story 1 of Epic 27 (the "moat" feature). It lands three things end-to-end: (1) the framework
primitive that lets one tool talk to a **second** profile (`ToolContext.resolveProfileClient`), (2) a new
ObjectScript endpoint that returns SHA-256 hashes of documents matching a spec (`GET/POST /dev/doc/hashes`),
and (3) the `iris_env_diff` tool wired for its **`documents`** domain only. Stories 27.1–27.4 add the other
diff domains, the promote plan/execute, docs, and the capstone.

## Acceptance Criteria

1. **AC 27.0.1** — `resolveProfileClient(profileName)` on `ToolContext` reuses the exact per-profile client construction/caching of the `server` param path (extracted, not duplicated); unknown profile throws naming known profiles; back-compat snapshot green (existing tools see no change — Rule #19).
2. **AC 27.0.2** — `$SYSTEM.Encryption` SHA API probed and pinned (Rule #16); `/dev/doc/hashes` endpoint returns `{name, hash, timestamp}` per matching doc; `spec` required (`*` refused with Rule #38 guidance unless `allowWide:true`); timeout risk documented; conventions §3 handler skeleton; `%UnitTest` coverage.
3. **AC 27.0.3** — `iris_env_diff` documents domain works end-to-end across two profiles: `onlyInSource`/`onlyInTarget`/`differs`(+hashes)/`identical` buckets; hash comparison idempotent and timestamp-insensitive under `ignoreTimestamps:true`.
4. **AC 27.0.4** — Bootstrap regen + BOOTSTRAP_VERSION from→to recorded (Rule #24).

### Integration AC (lead-side gate — this story is service-introducing)

This story introduces the framework primitive `resolveProfileClient` and the `/dev/doc/hashes` endpoint. **AC 27.0.3 IS the integration AC**: the consumer `iris_env_diff` (documents domain) reads from `resolveProfileClient` + the new endpoint and produces the observable diff buckets end-to-end across two profiles. No "no consumers yet" declaration is needed — the consumer ships in this same story.
