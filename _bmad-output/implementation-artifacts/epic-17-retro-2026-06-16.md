## What Went Well

- **Rule #16 pre-spec probing (Story 17.0) was the epic's MVP.** A dedicated probe story live-verified all three API areas BEFORE the implementation specs were finalized, catching two wrong API claims up front: `Ens.Config.Item.AdapterClassName` is a read-only calculated METHOD (not a settable property), and the `INFORMATION_SCHEMA` statement tables are UNDERSCORED (`CURRENT_STATEMENTS`/`STATEMENT_DAILY_STATS`, not the no-underscore forms). A third (`SaveToClass` writes XData, not the SQL extent → needs `LoadFromClass`) surfaced in 17.2 dev and was folded back into the probe doc (Rule #5 amendment).
- **Rule #19 mechanical back-compat gate (17.2) held and was strengthened.** The dev's request-body `toEqual` snapshots were strengthened by QA to FULL-OBJECT `toEqual` on the existing enable/disable/get/set output — a genuine fail-if-drift gate over an existing live tool. The lead smoke confirmed the 5 working property keys + add/remove/arbitrary-settings live (comment→property, `ArchivePath@Host`→Host Setting, `Charset@Adapter`→Adapter Setting).
- **Rule #24 per-story bootstrap regen worked cleanly.** `BOOTSTRAP_VERSION` moved incrementally per ObjectScript story (17.1, 17.2); 17.3 (TS/SQL-only) added nothing; 17.4 verified `gen:bootstrap` idempotent (no fresh diff) — exactly the reinterpretation routed from the Epic 16 retro, NOT a deferred single bump.
- **Rule #23/#25 frozen baseline held** — `1e62c5ad5bf7` (141 keys) unchanged across 2 new tools + 1 enhancement; new keys governed by `mutates`; `gen:governance-baseline:check` exit 0 every story; bare generator never run.
- **Live-HTTP lead smokes (Rule #26) confirmed two security guards live** that unit tests structurally cannot: the `||` IdKey-injection guard (17.1, rejected with no write) and the `add`-without-`className` rejection (17.2). All disposable `ZZZ*` targets cleaned up.
- **17.3 correctly determined Atelier/SQL-only** — no ObjectScript handler, no bootstrap contribution — keeping the closer's bump scoped to interop ObjectScript only.

## What Was Challenging

- **A governance misconception in the probe doc:** Story 17.0 wrote that sql-analyze's read actions need "no mutates." That would have thrown at registration — `assertGovernanceClassification` requires every non-baseline key (read OR write) to be classified. Caught at 17.3 spec-time (corrected to all-`read`), not at runtime, but it shows even a careful probe can carry a governance-contract gap. → **Rule #28.**
- **The Ens.Config extent-vs-XData split** cost an in-flight probe-doc amendment (17.2 DISCREPANCY #1b) and produced a confusing (but correct) `get`-after-`add` "not found" in the smoke. Reusable IRIS-API lesson. → **Rule #27.**
- **Composite-IdKey injection** was a latent quiet-corruption hole in 17.1 until the reviewer added the delimiter guard. → **Rule #29.**
