## Key Insights

1. **For an executable reference, observation beats reasoning.** The port author's mental model of the reference is exactly what's wrong when the port is wrong — so a hand-derived "expected" value tends to be mis-derived the same way. Run the reference; capture what it actually emits. (Rule #36.)
2. **Re-deferral is a valid per-epic decision but an invalid steady state.** The mechanism that let three feature epics stay focused (route-and-defer) is the same mechanism that let the ledger grow unbounded — the fix is a scheduled terminal-disposition cadence, not stricter per-epic gates. (Rule #37.)
3. **Output caps are not scan caps.** A `topN`/page-size parameter reads like a safety limit but bounds only the response, not the work — a real trap for any enumerate-then-truncate tool. The live 504 made it concrete. (Rule #38.)
