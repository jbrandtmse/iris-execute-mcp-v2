## 2. What Went Well

1. **The live capstone earned its keep — it caught a HIGH the mocked suite had encoded as correct.** Story 27.4's Lead-authorized seed→diff→plan→execute→re-diff round-trip exposed that `mappingKey` included the per-side namespace, so identical mappings across differently-named namespaces (HSCUSTOM vs SADEMO) never matched: spurious drift on every diff, and promote structurally unable to converge to clean. The mocked test suite was GREEN — its expected values had been derived from the implementation's own (buggy) output. Fixed (`type::name`), cross-namespace regression added + mutation-verified, capstone re-run clean. → codified as **Rule #49** and **Rule #50**.
2. **Adversarial Opus review earned 6 genuine security patches on the execute path.** Gate 3b plan-steps-vs-diff anti-tamper (all three layers), credential error-path scrub, fail-closed empty-document, Gate 3c wrong-instance. None were speculative — each had a concrete tamper/leak scenario.
3. **Probe-first specs (Rules #16/#42) held.** Story 27.0's probe pinned the doc-hash surface live (`$SYSTEM.Encryption.SHAHash` — `$ZCONVERT("O","HEX")` throws on this instance) before 27.1–27.3 coded against it.
4. **The safety architecture survived review and live fire.** NO-DELETE is type-enforced (`PlanOperation` closed union — no delete verb exists to reach); `onlyInTarget` → warnings; 4 refuse-before-any-write gates; `execute` default-disabled and NOT `defaultEnabled` (Rule #32 deliberately not invoked); frozen baseline untouched.
5. **Lead-directed mid-story reworks were cheap and correct.** All three (namespace routing, per-domain isolation + documents-opt-in, broadened redaction) were caught by lead smokes at the story boundary, not after ship.

## 3. What Could've Gone Better

1. **The mocked suite actively certified the bug.** Not just a coverage gap — the fixtures' expected values were implementation-derived, so the suite would have kept the bug green forever. The capstone was the ONLY thing standing between this HIGH and shipping. (→ Rules #49/#50.)
2. **The 27.0 spec under-specified per-profile namespace routing**; the first smoke returned `namespace:""`. Caught immediately (Rule #34-shape smoke), fixed in a rework loop. Also: the early "separate code DBs — drift IS seedable" assumption proved incorrect for documents (HSCUSTOM/SADEMO share code DBs) — another observe-don't-assume data point, absorbed by making `documents` opt-in.
3. **A bare `iris_env_diff` against a non-interop namespace originally aborted the whole diff** (no per-domain isolation, CR 27.1-3). The default-domain set and isolation semantics should have been in the 27.1 spec, not discovered by smoke.
4. **Windows `core.autocrlf` produced a recurring local-only `gen-skills-stray-file.test.ts` artifact** — noise each story; deferred note recorded (a `.gitattributes` would end it).

## 5. Rules Codified (→ `.claude/rules/project-rules.md`)

- **Rule #49 — Round-trip-to-clean live capstone for diff/sync/promote features; implementation-derived mocked fixtures encode the implementation's bugs.** For any feature whose contract is a comparison/synchronization invariant between two live sides, the epic-done gate must include a live seed→diff→plan→execute→re-diff-CLEAN round-trip; convergence-to-clean is a self-validating oracle no mocked fixture provides.
- **Rule #50 — A comparison/matching key must contain only item-identity dimensions, never a per-side dimension.** Namespace was the *side*, not the item's identity; a side dimension in the key makes cross-side matches structurally impossible. Litmus: "would two genuinely-identical items on the two sides differ in this component?" — if yes, remove it from the key (compare it as value instead).

**Candidate NOT codified:** "per-profile namespace routing" — narrow one-off (fixed in 27.0 with regression tests); the smoke discipline that caught it is already Rules #34/#40. Per Rule #1, it stays in the story record.
