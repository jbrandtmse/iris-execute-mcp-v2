## 2. What Went Well

1. **The mandatory probe (28.0) paid for itself immediately.** It empirically caught that `%Dictionary.CompiledStorage.ExtentSize` / `CompiledStorageProperty.Selectivity` are declarative/compile-time-only and do NOT reflect live `TuneTable` state on 2026.1 (verified 4 ways incl. post-recompile + direct `SetExtentSize`). `stale-stats` was re-sourced to the EXPLAIN plan's own `Warning: Table X is not tuned.` line — simpler and zero-extra-call. A wrong heuristic source was corrected in the spec BEFORE 28.2 built on it (Rule #16/#42 working as intended).
2. **The split architecture held.** OS endpoint returns raw materials; the heuristic engine is pure TS, fixture-testable, with NO bootstrap bump per heuristic tweak. 46 engine tests + 3 further review/QA tests all replay reference-captured JSON.
3. **Rule #36 reference-capture discipline was airtight end-to-end.** Every fixture cited its live capture command + IRIS version; the Opus 28.2 reviewer spot-re-captured two fixtures live and confirmed byte-for-byte match (incl. the `.Master`-suffix and bare-column-predicate variances the dev discovered mid-capture and folded into the parser rather than assuming).
4. **The engine derived table identity from PLAN TEXT, not the endpoint's extracted `tables` list** — a deliberate design choice to insulate it from the deferred 28.1 `ResolveClassForTable`/`ExtractTables` hazards (CR 28.1-1/-2). The reviewer confirmed this held.
5. **Three Rule #47 spec corrections recorded** (no existing OS handler for `iris_sql_analyze`; `ExecuteMCPv2.Tests` plural; `Temp.*` probes) — each verified true against source, preventing 28.1–28.3 from coding against a stale "read the existing handler" instruction.
6. **Every per-story lead smoke was genuinely live** — HSCUSTOM missing-index DDL, the full HTTP endpoint envelope, the e2e endpoint→built-engine pipeline, and the Rule #34 SADEMO second-namespace routing + a 2-table join (CR 28.2-1 path exercised live).

## 3. What Could've Gone Better

1. **The `.gitattributes` ride-along was incomplete at kickoff.** Adding `* text=auto eol=lf` fixes FUTURE checkouts but does NOT convert already-checked-out CRLF files; the `skills/**` files still failed `gen-skills --check` (a raw-read validator) until a one-time LF renormalization during Story 28.1. Now fully resolved (`@iris-mcp/all` 49/49). *Lesson recorded (not a rule — the Project Lead judged it a narrow hygiene gotcha): when adding eol normalization on a Windows/autocrlf checkout, follow it with a one-time renormalize so raw-read `--check` validators go green.*
2. **The code-review skill's async layers returned empty on first pass in one review (28.1)** — a harness flake requiring a coordinator nudge to collect the Blind/Edge/Auditor findings; cost a round-trip. Mitigated in later reviews by an explicit "wait for all three layers before ending your turn" instruction in the spawn prompt.
3. **Two MED heuristic defects were latent in the first engine draft** (the harmful `unused-index` "drop your primary key" false-positive on system classes where the plan map-name differs from the dictionary index name; a throw on malformed `indexes` shapes). Both caught + patched + mutation-verified at the 28.2 review — the adversarial review did its job, but they indicate the first-draft engine under-handled system-schema and malformed-shape edges.
